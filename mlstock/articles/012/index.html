Machine Learning for Stock Markets

Knowledge Base



Testing Stock Market Efficiency Using Historical Trading Data and Machine Learning



Autori

SAMI PURMONEN, PAUL GRIFFIN

Anno

2015

In breve

Verificare la teoria economica Efficient Market Hypotesis, ovvero che il mercato contiene già tutte le info che lo riguardano e che quindi esprime già il suo massimo.

Metodi

  • Nearest Neighbour classifier
  • Neural Network
  • Random Forest classifier
  • Linear regression
  • Naive (algoritmo dummy usato per comparare i risultati, nessun apprendimento)

Metriche

  • Absolute Percentage Error (the difference between the real values and predicted values).
  • Movement accuracy (the measurement is the percentage of predicted movements that are correct).
  • Investment profit (the value represents using the output of one of the predictors to buy stocks).

Dataset

Dati di mercato giornalieri per gli indici S&P500 e FTSE.

Risultati

Rispetto alle metriche utilizzate, questi sono i risultati dell’esperimento:

  • Absolute Percentage Error: gli algoritmi di ML producono errori <1%. Tuttavia, siccome l’algoritmo Naive si comporta decisamente meglio, la conclusione è che ML non offre vantaggi nella predizione del valore di mercato.
  • Movement accuracy: gli algoritmi di ML hanno una precisione dell’ordine di 50%, comunque meno di Naive a 56%.
  • Investment profit: sebbene tutti gli algoritmi di ML abbiano un ritorno positivo su S&P500, l’algoritmo Naive (che si limita a seguire l’indice) ha risultati migliori. Inoltre, se venissero considerate le fee di transazione, le simulazioni avrebbero esito negativo.

La conclusione è che nessuno degli algoritmi di machine learning proposti ha ottenuto prestazioni migliori rispetto all’algoritmo Naive che non implementa tecniche di apprendimento ma si limita a seguire il trend del mercato.
